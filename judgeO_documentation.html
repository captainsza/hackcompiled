Judge0 Documentation

About

Judge0 is a robust, scalable, and open-source online code execution system. It can be used to build a wide range of applications that require online code execution features, such as competitive programming platforms, e-learning platforms, candidate assessment and recruitment platforms, online code editors, online IDEs, and more.

In our research paper Robust and Scalable Online Code Execution System, we present Judge0's modern modular architecture that can be easily deployed and scaled. We discuss its design, address various challenges in building such systems, and compare it with other available online code execution systems and online judge systems.

To see Judge0 in action, try Judge0 IDE, our free and open-source online code editor. You can also use a dummy client to explore and test most of Judge0's features.

Features

  • Quick and easy installation
  • Rich and verbose API documentation
  • Scalable architecture
  • Sandboxed compilation and execution
  • Support for over 60 programming languages
  • Compilation and execution of multi-file programs
  • Support for additional files alongside the user's program
  • Support for custom user-defined compiler options, command-line arguments, and time and memory limits
  • Detailed execution results
  • Webhooks (HTTP callbacks)

Authentication

Administrators of Judge0 can configure it to require an authentication token (API key). If required, provide the X-Auth-Token header field in every API request.

Authenticate

POST /authenticate

Check if your authentication token is valid.

Note:

  • X-Auth-Token is the default header field name, but administrators may change it.
  • Contact the administrator of the Judge0 instance to obtain your authentication token.

Security Warning: Always send the authentication token through headers, not as a URI parameter.

Example URI:

POST https://ce.judge0.com/authenticate

Authorization

Some API calls require authorization. For example, listing all submissions on Judge0 requires authorization.

Authorize

Check if your authorization token is valid. If authentication is enabled, you must also authenticate in this API call.

Note:

  • X-Auth-User is the default header field name, but administrators may change it.
  • Contact the administrator to obtain your authorization token.

Security Warning: Always send the authorization token through headers, not as a URI parameter.

Example URI:

POST https://ce.judge0.com/authorize

Single-File and Multi-File Programs

Judge0 supports both single-file and multi-file programs in a sandboxed environment with specified resource limitations.

Single-File Programs

The simplest way to run a program with Judge0. Select a language_id, provide source_code, and Judge0 uses predefined compilation and execution scripts. You can set custom compiler flags, provide command-line arguments, and include additional_files (e.g., Python modules or C/C++ header files) available in the sandbox.

Multi-File Programs

Available since v1.10.0, multi-file programs allow custom compilation and execution scripts. Use language ID 89 ("Multi-file program") and send all program files via the additional_files attribute as a Base64-encoded zip archive. The source_code attribute cannot be used. Include compile and run Bash scripts in the zip's root to define compilation and execution steps.

Example: For a multi-file C++ project using CMake, include a compile script for compilation and a run script for execution.

Create a Submission

POST /submissions/{base64_encoded_wait}

Creates a new submission that waits in a queue to be processed. Returns a submission token to check status.

Example URI:

POST https://ce.judge0.com/submissions/?base64_encoded=false&wait=false

URI Parameters:

  • base64_encoded (boolean, optional, default: false): Set to true to send Base64-encoded data.
  • wait (boolean, optional, default: false): Set to true to get submission results immediately (if enabled).

Note: Using wait=true is not recommended as it does not scale well.

Response Example:

{
  "token": "d85cd024-1548-4165-96c7-7bc88673f194",
  "base64_encoded": false,
  "set_to_true": "stdout time memory etdrtoken compile output"
}

Get a Submission

GET /submissions/{?base64_encoded,fields,page,per_page}

Example URI:

GET https://ce.judge0.com/submissions/?base64_encoded=false&fields=stdout,time,memory,stderr,token,compile_output&page=4&per_page=2

Delete a Submission

DELETE /submissions/{token}?fields=stdout,stderr,status,id,language_id

Requires authorization. Deletion may be restricted by the Judge0 instance administrator.

Example URI:

DELETE https://ce.judge0.com/submissions/d85cd024-1548-4165-96c7-7bc88673f194?fields=stdout,stderr,status,id,language_id

URI Parameters:

  • token (string, required): Submission token obtained during creation.
  • fields (string, optional): Desired attributes to return (e.g., stdout, stderr).

Submission Batch

POST /submissions/batch?{base64_encoded}

Create multiple submissions at once.

Example URI:

POST https://ce.judge0.com/submissions/batch?base64_encoded=false

URI Parameters:

  • base64_encoded (boolean, optional, default: false): Set to true for Base64-encoded data.

Get a Submission Batch

GET /submissions/batch/{tokens,base64_encoded,fields}

Example URI:

GET https://ce.judge0.com/submissions/batch?tokens=db54881d-bcf5-4c7b-a2e3-d33fe7e25de7,ecc52a9b-ea80-4400-ad50-4ab6cc3bb2a1,1b35ec3b-5776-48ef-b646-d5522bde2ccc&base64_encoded=false&fields=token,stdout,stderr,status,id,language_id

URI Parameters:

  • tokens (string, required): Comma-separated submission tokens.
  • base64_encoded (boolean, optional, default: false): Set to true for Base64-encoded data.
  • fields (string, optional): Desired attributes to return.

System Info

Judge0 consists of two systems: web and worker. The web system processes submissions and may be hosted on different machines. This API returns information from the web system.

GET /system_info

Example URI:

GET https://ce.judge0.com/system_info

Contact the Judge0 instance administrator to determine if web and worker systems are on the same host.

Configuration Info

GET /config_info
